# 콘텐츠 품질 체크리스트

`inblog-write-seo-post` 워크플로우의 Phase 4에서 발행 전 확인합니다.

---

## 1. 색인 필수 조건 (하나라도 빠지면 색인 실패 리스크)

- [ ] 콘텐츠 길이 3,000자+ (한국어 기준)
- [ ] 이미지 3장+ (모든 이미지에 alt 텍스트 포함)
- [ ] 내부 링크 2개+ (앵커 텍스트에 타겟 키워드 포함)
- [ ] meta_description 설정 (150-160자, 키워드 + 가치 제안)
- [ ] 커버 이미지 (image 필드) 설정 (1200x630px 권장)

## 2. 랭킹 경쟁력 (상위 노출을 원하면)

- [ ] 경쟁 키워드: 5,000자+ 콘텐츠
- [ ] meta_title 설정 (title과 다른 SERP 최적화 버전, 60자 이내)
- [ ] author_ids로 저자 연결 (E-E-A-T: 전문성 신호)
- [ ] JSON-LD 구조화 데이터 (Article 또는 HowTo 스키마)
- [ ] 4종류 이상의 콘텐츠 블록 타입 사용 (p, list, code, table, callout 등)
- [ ] 외부 권위 있는 사이트 링크 2개+ (출처 명시)
- [ ] H2→H3→H4 계층 구조 (H3만 사용 금지, 질문형 헤딩 권장)

## 3. AI 인용 가능성 (Citability)

AI 검색 플랫폼(ChatGPT, Perplexity, Gemini 등)이 콘텐츠를 인용하려면 **추출 가능한 구조**가 필요합니다.

### Answer-First 구조
- [ ] 각 H2 섹션의 **첫 1-2문장이 핵심 답변** (설명은 그 뒤에)
- [ ] 정의 패턴 사용: "X는 [정의]이다" / "X란 [설명]을 말한다"
- [ ] 구체적 수치/데이터 포함 ("평균 50% 향상", "3가지 방법" 등)

### 문단 구조
- [ ] 문단당 2-4문장 (1문장 문단 = 약함, 5문장+ = 추출 어려움)
- [ ] **한 문단 = 한 아이디어** (주제 혼합 금지)
- [ ] 각 문단이 **독립적으로 추출해도 의미 전달** 가능
- [ ] 핵심 주장을 문단 **첫 문장에** 배치

### 비교/리스트 구조
- [ ] 비교 콘텐츠: 테이블 형태로 정리 (AI가 테이블을 잘 인용)
- [ ] 절차 콘텐츠: 번호 매긴 순서형 리스트 사용

## 4. E-E-A-T 시그널 (콘텐츠 신뢰도)

### Experience (경험)
- [ ] 1인칭 경험 언급 ("직접 테스트해본 결과...", "우리 팀에서 적용한 사례...")
- [ ] 구체적 사례/결과 수치 포함 (추상적 조언이 아닌 실제 데이터)
- [ ] 스크린샷, 도표 등 직접 경험의 증거

### Expertise (전문성)
- [ ] 주제에 적절한 기술적 깊이 (피상적 설명 지양)
- [ ] 데이터 기반 주장 (통계, 연구 인용)
- [ ] 전문 용어의 정확한 사용 (처음 사용 시 정의 포함)

### Authoritativeness (권위)
- [ ] 저자 프로필 연결 (author_ids)
- [ ] 외부 권위 있는 출처 인용 (공식 문서, 연구 논문, 업계 리포트)

### Trustworthiness (신뢰)
- [ ] 주장에 대한 출처 명시 (링크 또는 인용)
- [ ] 트레이드오프 인정 ("이 방법은 X에 좋지만, Y 상황에서는 Z가 더 적합")
- [ ] 업데이트 날짜가 명확한 시점 정보 (published_at 설정)

## 5. AI 콘텐츠 품질 (고품질 시그널)

### 고품질 시그널 (있어야 함)
- [ ] **독자적 데이터/인사이트**: 다른 곳에서 얻을 수 없는 정보
- [ ] **구체적 예시**: 이름, 숫자, 날짜가 있는 사례 (추상적 설명 X)
- [ ] **전문적 판단**: 단순 사실 나열이 아닌 의견/권장 사항
- [ ] **실행 가능한 조언**: 모호한 가이드가 아닌 구체적 액션

### 저품질 시그널 (없어야 함)
- [ ] ~~"오늘날 빠르게 변화하는 세상에서..."~~ 같은 AI 특유 도입부 **없음**
- [ ] ~~"주목할 만한 점은..."~~, ~~"흥미로운 점은..."~~ 같은 무의미한 전환 **없음**
- [ ] 모든 섹션이 동일한 패턴으로 끝나는 반복 **없음**
- [ ] "다양한 요인에 따라 달라집니다" 같은 구체성 없는 헤징 **없음**
- [ ] 삭제해도 정보 손실 없는 필러 문단 **없음**

## 6. 파생 콘텐츠 추가 조건

외부 소스를 기반으로 작성한 콘텐츠인 경우:

- [ ] 원본 대비 50% 이상 독자적 콘텐츠
- [ ] canonical_url 설정 여부 판단 (충분히 독자적이면 미설정)
- [ ] 원본과 **다른** heading 구조
- [ ] 한국 시장 데이터/사례 또는 독자적 분석 포함
- [ ] 출처 callout + bookmark 링크로 원본 명시

## 7. 콘텐츠 신선도 (Freshness)

- [ ] published_at 설정 (구체적 날짜, "최근" 같은 모호한 표현 X)
- [ ] 시간 의존적 정보에 연도/날짜 명시 ("2026년 기준", "3월 현재")
- [ ] "최신", "현재", "올해" 같은 표현 대신 구체적 시점 사용

## 8. 포스트 메타데이터 최종 체크

- [ ] title (60자 이내, 주요 키워드 앞쪽)
- [ ] meta_title (title과 다른 SERP 최적화 버전)
- [ ] meta_description (150-160자, 키워드 + 가치 제안 + 행동 유도)
- [ ] slug (소문자+하이픈, 불용어 제거, 키워드 포함)
- [ ] image 필드 (커버/OG 이미지, 1200x630px)
- [ ] author_ids 연결
- [ ] tag_ids 연결 (3-5개)
- [ ] cta_text + cta_link (글 주제에 맞게 커스터마이즈)

## 9. 시리즈 포스트 반복 방지

동일 시리즈/캠페인의 포스트인 경우:

- [ ] 동일 템플릿 구조 3회 이상 반복 금지
- [ ] 고정 섹션명 ("인블로그 팀 코멘트" 등) 글마다 변경
- [ ] CTA 문구 글별 커스터마이즈 (5개 포스트에 동일 CTA 금지)
- [ ] 문장 구조 변화 (모든 문장이 평서문이면 안 됨)
